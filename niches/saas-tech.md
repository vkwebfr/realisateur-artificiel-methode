# Niche : SaaS & Tech

Prompts et modèles prêts à l'emploi pour les vidéos de logiciels SaaS, applications, plateformes B2B, outils IA, dashboards et démonstrations de produits numériques.

---

## Contexte de la niche

Une vidéo SaaS ne doit pas seulement montrer une interface. Elle doit rendre une transformation visible. Le spectateur doit comprendre rapidement quel problème disparaît, quelle tâche devient plus simple et quel résultat devient possible.

Les émotions qui vendent dans cette niche sont la clarté, l'efficacité, le soulagement, la maîtrise et l'effet de surprise. Le produit doit sembler facile à utiliser, même si la technologie derrière est complexe.

Les codes visuels qui fonctionnent : interface propre, dashboard lisible, curseur précis, clics visibles, données qui apparaissent progressivement, automatisations, raccourcis clavier, transitions fluides entre écrans, détails de clavier, lumière de bureau premium et résultats avant/après.

Ce qui ne fonctionne pas : photos de stock de personnes pointant un écran, gradients bleus génériques, lignes de code aléatoires, animations trop rapides, interfaces illisibles, jargon technique sans démonstration et accumulation de fonctionnalités sans hiérarchie.

Marques et références à étudier : Notion pour la simplicité, Linear pour la fluidité, Stripe pour la clarté, Apple pour la mise en scène du produit, Slack pour les cas d'usage, Figma pour la collaboration et les meilleures campagnes de logiciels IA pour l'effet de transformation immédiate.

## 5 briefs prêts à l'emploi

### Brief 1 — Le problème disparaît

Vidéo de 20s pour présenter [NOM DU LOGICIEL]. On commence par une tâche lente, confuse ou répétitive. L'interface apparaît, automatise le processus et transforme le chaos en résultat clair. Ambiance : efficacité, soulagement, maîtrise. Palette : interface sombre ou claire, accent de marque, lumière de bureau douce. Référence : Linear x Stripe. Format : 9:16.

### Brief 2 — La démonstration en trois clics

Vidéo de 15s montrant comment [NOM DU LOGICIEL] permet d'obtenir [RÉSULTAT] en trois actions simples. Chaque clic doit produire une conséquence visible : importation, analyse, génération, validation. Le spectateur doit comprendre le bénéfice sans lire un long texte. Ambiance : rapide, précise, satisfaisante. Référence : Notion. Format : 9:16.

### Brief 3 — Le workflow automatisé

Vidéo de 25s pour présenter une automatisation dans [NOM DU LOGICIEL]. On visualise les étapes du workflow : déclencheur, traitement, synchronisation, notification, résultat final. L'interface doit sembler fluide et fiable. Ambiance : contrôle, tranquillité, gain de temps. Palette : graphite, blanc, accent vert ou violet. Format : 16:9 ou 9:16.

### Brief 4 — Le dashboard qui donne la réponse

Vidéo de 20s centrée sur un dashboard de [NOM DU LOGICIEL]. On commence avec des données brutes, puis les graphiques, indicateurs et recommandations apparaissent progressivement. Le dernier plan montre une décision claire prise grâce au produit. Ambiance : intelligence, lisibilité, confiance. Référence : Stripe Dashboard x Tableau premium. Format : 16:9.

### Brief 5 — La collaboration sans friction

Vidéo de 20s pour montrer comment une équipe utilise [NOM DU LOGICIEL]. Un document, une tâche ou un projet passe d'une personne à l'autre sans email inutile. Commentaires, validation, statut et résultat final sont visibles dans la même interface. Ambiance : coordination, fluidité, efficacité collective. Référence : Notion x Figma. Format : 9:16 ou 16:9.

## Pièges courants dans cette niche

* Montrer trop de fonctionnalités dans une seule vidéo : un film doit vendre un bénéfice principal.
* Utiliser une interface trop petite : les informations importantes doivent rester lisibles sur mobile.
* Ajouter des lignes de code sans fonction narrative : cela crée du bruit, pas de la crédibilité.
* Utiliser des personnes de stock qui pointent un écran : montrer le produit en action.
* Employer un gradient bleu générique : respecter l'identité réelle de la marque.
* Accélérer excessivement le curseur : chaque action doit rester compréhensible.
* Inventer une interface impossible : les écrans doivent correspondre au produit réel ou à un prototype validé.
* Afficher trop de texte : privilégier les actions, résultats, chiffres clés et transformations visuelles.
* Montrer un dashboard sans expliquer la décision qu'il permet de prendre.
* Utiliser des effets futuristes exagérés pour un logiciel simple : la clarté vend mieux que le spectacle artificiel.
